# UI_SPEC.md — THE HYUNDAI POS (Restaurant + FoodCourt)

> 목표: 아래 UI 규칙을 **최우선(불변)** 으로 준수한다.
> 단, 비즈니스 로직/데이터는 별도 스펙(PROMPT/INTEGRATION)에 따르며,
> UI는 Compose(Material3)로 구현한다.

---

## 0) 공통 규칙

### 0.1 화면 기본 레이아웃
- 기본 화면은 **가로(태블릿/데스크탑) 기준**으로 설계한다.
- 대부분 화면은 **2-pane** 구조를 사용한다.
  - Restaurant: (좌) 테이블 영역 / (우) 주문 패널
  - Food: (좌) 주문 리스트 / (우) 메뉴 선택 영역 + (하단) 고정 액션바

### 0.2 상단 고정 탑바 (모든 화면 공통)
- 좌측: **"THE HYUNDAI"** 브랜드 텍스트
- 좌측 상단 근처: 날짜/시간 표시
- 중앙: 칩 형태로
  - `포스 : ####`
  - `거래 : ####`
- 우측 버튼(일렬, 항상 노출):
  - `점검`
  - `조회`
  - `영수증 재출력`
  - `더보기`
- 탑바는 스크롤과 무관하게 **항상 고정**.

### 0.3 스타일 톤
- 배경은 밝은 톤(화이트/라이트 그레이 기반).
- 주요 강조 색:
  - 딥그린(테이블 선택/활성 패널 헤더)
  - 베이지(보류/특정 상태)
- UI는 **정보 밀도 높게**, 버튼은 큼직하게(터치 최소 48dp).

## 1) Restaurant (테이블 관리 화면)

### 1.1 상단 Area 탭
- 탑바 바로 아래: Area 탭(예: `식당가 1층 홀`, `식당가 1층 룸`, `식당가 2층 홀`, `야외 테라스`)
- 선택된 탭은 하단 인디케이터(딥그린 계열).

### 1.2 본문 2-pane
- **좌(메인)**: 테이블 배치/그리드 영역
- **우(고정 패널)**: 선택 테이블 주문 패널 (고정 폭, 스크롤 독립)

#### 1.2.1 좌측 테이블 영역 (Table Layout)
- 테이블은 카드/타일로 표현하되 시안처럼 “배치된 느낌”을 주는 여백/간격 사용.
- 테이블 타일 종류:
  - **사각 타일**: 기본 테이블
  - **원형 타일**: 대표/큰 테이블(예: T-4)
- 테이블 타일 표시 요소(중앙 정렬):
  - 1줄: 테이블명(예: `T-1`)
  - 2줄: 금액(예: `45,000원`)
  - 3줄: 경과시간(예: `35분`) + 인원(예: `2명`) (아이콘은 선택)
- 상태 표현(색/스타일):
  - EMPTY(빈자리): 흰색/연한 배경 + "빈자리" 텍스트
  - OCCUPIED(식사중): 딥그린 배경 + 흰 텍스트(금액 강조)
  - BILLING/보류/기타: 베이지/그레이 계열 배경(시안의 베이지 타일 느낌)
  - DISABLED: 비활성(회색) + 클릭 불가
- 테이블 선택 시:
  - 선택 테두리/그림자/강조 표시(딥그린 계열)
  - 우측 패널이 해당 테이블로 갱신됨

#### 1.2.2 우측 주문 패널 (Order Side Panel)
- 패널 헤더(상단 고정):
  - 딥그린 배경
  - 큰 글씨로 테이블명(예: `T-1`)
  - 보조 정보: `식사중 35분 | 2/4명`
- 헤더 아래: 주문 아이템 리스트
  - 한 줄 구성: `상품명` (좌) / `수량` (우)
  - 구분선으로 행을 구분
  - 스크롤 가능
- 패널 하단 고정 영역:
  - `총 주문금액` 라벨 + 금액(우측 정렬, 강조 색)
  - 큰 버튼 `결제` (패널 하단 폭 가득, 높이 큼)

> Restaurant 화면에서는 "주문 추가" 버튼은 필수 아님.
> 단, 구현 편의상 More/더보기 메뉴 또는 패널 내 보조 버튼으로 추가할 수 있음.

## 2) Food (푸드코트 주문 화면)

### 2.1 본문 2-pane + 하단 고정 액션바
- **상단 탑바 고정**
- 본문은 좌/우 2-pane
- 화면 맨 아래: **하단 고정 액션바(3버튼)**

### 2.2 좌측 패널: 현재 주문 리스트(카트)
- 좌측 상단: 현재 테이블/주문 컨텍스트 표시 (예: `Table 1`)
- 그 아래: 리스트 헤더(열 구성)
  - `상품명` | `수량` | `금액`
- 리스트 행:
  - 상품명
  - 수량
  - 금액
  - 삭제(X) 아이콘(우측)
- 좌측 패널 하단:
  - `받을 금액` 라벨
  - 총액(큰 글씨, 빨간색 강조)

### 2.3 우측 영역: 카테고리 탭 + 메뉴 그리드
- 우측 상단: 카테고리 탭(시안처럼 가로 탭)
  - 예: `식사중`, `오모&커리`, `본까스`, `해천죽`, `이십사절기`, `미고`, `도레도레`
- 탭 아래: 메뉴 카드 그리드(바둑판)
  - 카드 구성: 상단에 메뉴명, 하단에 가격
  - 카드 클릭 시 좌측 주문 리스트에 추가(+1)
  - 품절인 경우:
    - 클릭 불가
    - "품절" 표시(에러 컬러)

### 2.4 하단 고정 액션바 (3 버튼)
- 화면 최하단에 고정, 본문 스크롤과 분리
- 버튼 3개가 가로로 동일 비율 배치:
  1) `반품/환불` (회색/중립 톤)
  2) `주문 보류` (딥그린 톤)
  3) `결제 진행` (베이지/강조 톤)
- 동작(네비게이션):
  - `결제 진행` 클릭 → Cart/Confirm 화면으로 이동 (tableId 유지)
  - `반품/환불`, `주문 보류`는 MVP에선 TODO 가능(버튼/레이아웃은 반드시 존재)

## 3) 데이터 표시 규칙(시안 연동)
- Restaurant 테이블 타일의 금액/경과시간은 **DB(실제 주문) 기반**으로 표시한다.
- Food 좌측 주문 리스트는 **공유 카트 상태**(Food ↔ Cart/Confirm)로 유지한다.
- 합석/이동 등으로 테이블이 바뀌면 Restaurant 우측 패널/타일 정보가 즉시 반영되어야 한다.

---

## 4) 금지 사항
- 화면을 단일 컬럼 모바일 UX로 단순화하지 않는다(시안은 2-pane 중심).
- 탑바/우측 주문 패널/하단 액션바를 스크롤에 묻히게 만들지 않는다(고정).
- 버튼 텍스트/배치/역할을 임의 변경하지 않는다.

## 5) Super POS 공통 디자인 원칙

### 5.1 디자인 언어
- Super POS 화면은 기존 Restaurant / Food 화면의 디자인 언어를 그대로 계승한다.
- 공통 톤:
  - 상단 고정 `PosTopBar`
  - 라이트 그레이/라이트 베이지 계열 배경
  - 딥그린(`PRIMARY_GREEN`) 강조색
  - 베이지(`ACCENT_BEIGE`) 주요 CTA 색상
  - 블루그레이(`ACCENT_SLATE`) 보조 강조색
- 모든 액션 버튼은 크게, 넓게, 터치 친화적으로 만든다.
- 카드/타일/버튼/패널은 둥근 모서리와 충분한 여백을 사용한다.
- 화면은 Android POS/PDA 가로 화면 기준으로 설계한다.

### 5.2 네비게이션
- `SuperHome` 는 슈퍼 POS 홈 화면이다.
- `ProductRegister` 는 **판매용 상품등록 화면**이다.
- 여기서 `ProductRegister` 는 "상품 마스터 등록 폼"이 아니라,
  "상품을 장바구니에 담고 결제/시재로 이어지는 판매 등록 화면"을 의미한다.
- Navigation flow:
  - `SuperHome -> ProductRegister()`
  - `SuperHome -> ProductRegister(barcode)`
- 홈 화면에서 바코드 스캔 성공 시 `ProductRegister(barcode=...)` 로 이동한다.
- 홈 화면에서 `상품등록` 버튼/타일 클릭 시 `ProductRegister()` 로 이동한다.

## 6) SuperHome 화면

### 6.1 목적
- 슈퍼 POS의 메인 홈/대기 화면
- 바코드 스캔 진입점
- 판매용 상품등록 화면 진입점
- 자주 쓰는 주요 기능 빠른 접근
- 최근 거래/직전 결제 상태 확인

### 6.2 레이아웃 구조
- 상단: 공통 `PosTopBar`
- 탑바 아래:
  1) 바코드 스캔/입력 바
  2) 메인 콘텐츠 2-pane
  3) 하단 고정 유틸리티 바

### 6.3 바코드 스캔/입력 바
- 화면 상단 중앙에 긴 스캔/입력 바를 배치한다.
- placeholder:
  - `바코드를 스캔하세요`
- 하드웨어 스캐너가 keyboard wedge 방식이면:
  - TextField 포커스를 유지해야 한다
  - 입력 완료(Enter/개행) 시 `ProductRegister(barcode)` 로 이동한다
- 카메라 스캔을 쓰는 경우도 동일하게 스캔 성공 시 `ProductRegister(barcode)` 로 이동한다

### 6.4 메인 콘텐츠
- 메인 콘텐츠는 좌/우 2-pane
- 권장 비율:
  - 좌 65%
  - 우 35%

#### 6.4.1 좌측 메인 액션 영역
좌측에는 큰 Hero Tile Row + 보조 기능 Grid를 배치한다.

##### A. Hero Tile Row
- 큰 타일 3개 가로 배치
- 기본 항목:
  1) `상품판매`
  2) `상품등록`
  3) `통합조회`
- 타일 색상:
  - `상품판매`: 딥그린
  - `상품등록`: 베이지/웜톤
  - `통합조회`: 슬레이트/딥네이비
- `상품등록` 클릭 시 `ProductRegister()` 이동

##### B. Secondary Action Grid
- Hero Tile 아래 2차 기능 버튼 그리드
- 기본 3열
- 예시:
  - `즐겨찾기`
  - `행사보관`
  - `영수증 조회`
  - `소비기한 등록`
  - `상품조회`
  - `포인트 조회`
  - `보류/복원`
  - `도중회수`

#### 6.4.2 우측 상태 패널
- `직전결제내역` 또는 `최근 거래 요약` 카드 배치
- 표시 항목 예시:
  - 상품수량
  - 구매금액
  - 할인금액
  - 받은금액
  - 거스름돈
- 하단 바로가기 예시:
  - `직전 영수증 출력`
  - `직전 거래 환불`
  - `보류/복원`
  - `도중회수`

### 6.5 하단 고정 유틸리티 바
- 홈 화면 최하단에는 고정 유틸리티/도구 바를 둔다
- 예시:
  - `고정 메뉴`
  - `시재 점검`
  - `셀프계산대 전환`
  - `전달노트`
  - `따라하기`
  - `직원외 할인`

## 7) ProductRegister 화면 (판매용 상품등록 화면)

### 7.1 목적
- 바코드 스캔 또는 화면 터치로 상품을 장바구니에 등록한다
- 좌측 장바구니/합계 확인
- 우측 카테고리/상품 선택
- 하단 결제/시재 관련 액션 진입

### 7.2 개념 정의
- 이 화면은 **판매할 상품을 등록하는 POS 주문/판매 화면**이다
- 상품 마스터 관리용 등록 폼이 아니다
- 장바구니에 담긴 상품을 기준으로:
  - 할인
  - 보류
  - 취소
  - 시재 선택
  - 결제 진입
  을 수행한다

### 7.3 레이아웃 구조
- 상단: 공통 `PosTopBar`
- 본문: 좌/우 2-pane
- 하단: 좌측 요약 영역 + 우측 액션 버튼 영역
- 권장 비율:
  - 좌측 45%
  - 우측 55%

### 7.4 좌측 패널: 장바구니/상품등록 리스트
좌측 패널은 현재 담긴 상품 목록과 결제 요약을 표시한다.

#### 7.4.1 리스트 헤더
- 컬럼 헤더:
  - `No.`
  - `상품`
  - `단가`
  - `수량`
  - `금액`

#### 7.4.2 리스트 행 구성
- 한 행은 다음 정보를 가진다:
  - 순번
  - 상품명
  - 상품 보조정보(예: 바코드, 옵션, 분류 등) 2번째 줄 표시 가능
  - 단가
  - 수량 조절 컨트롤
  - 금액
  - 삭제 버튼(X)
- 수량 조절은 원형 버튼/터치 큰 버튼 사용:
  - `-`
  - 현재 수량
  - `+`
- 삭제 버튼은 우측 끝에 크게 배치
- 리스트는 스크롤 가능

#### 7.4.3 좌측 하단 액션 버튼
- 리스트 하단에는 다음 3개 버튼을 가로 배치한다:
  1) `행사적용`
  2) `주문 보류`
  3) `전체취소`
- 스타일:
  - `행사적용`, `주문 보류`: 외곽선/화이트 계열 버튼
  - `전체취소`: 외곽선 + 경고 텍스트 강조
- 버튼은 충분히 크게 만든다

#### 7.4.4 금액 요약 영역
- 좌측 최하단에 정산 요약 표시
- 기본 항목:
  - `총 매출`
  - `할인금액`
  - `받을 금액`
- `받을 금액`은 가장 크게 강조
- 컬러:
  - 기본 수치: 진한 회색/검정
  - 할인금액: 블루 강조 가능
  - 최종 받을 금액: 레드 강조

### 7.5 우측 패널: 카테고리 + 상품 그리드 + 시재/결제 액션

#### 7.5.1 카테고리 탭 바
- 우측 상단에는 가로 카테고리 탭 바 배치
- 예시:
  - `즐겨찾기`
  - `채소`
  - `과일`
  - `생수`
  - `종량제 봉투`
  - `기타`
- 선택된 탭은 하단 인디케이터 + 강조색 적용
- `즐겨찾기`는 별 아이콘 사용 가능

#### 7.5.2 상품 카드 그리드
- 탭 아래에 상품 선택 그리드
- 기본 4열 카드 그리드
- 각 카드 표시:
  - 상품명
  - 가격
- 카드 클릭 시 장바구니에 상품 1개 추가
- 품절 상품은 클릭 불가 + 품절 표시 가능
- 상품이 많으면 스크롤 가능

#### 7.5.3 우측 하단 액션 영역
- 우측 하단에는 2행 x 3열 큰 액션 버튼 배치
- 기본 버튼:
  1) `장바구니할인`
  2) `상품권`
  3) `현금`
  4) `기타시재`
  5) `H.Point 사용`
  6) `카드/모바일`
- 색상 그룹:
  - `장바구니할인`, `기타시재`: 블루그레이
  - `상품권`, `H.Point 사용`: 딥그린
  - `현금`, `카드/모바일`: 베이지 강조
- 버튼은 동일 크기, 큰 사각형, 중앙 정렬
- 이 액션 영역은 화면 하단에 고정되어야 한다

### 7.6 바코드 스캔 동작
- 이 화면에서도 하드웨어 스캐너 입력을 받을 수 있어야 한다
- visible input 없이도 스캔 이벤트를 받아 상품을 장바구니에 추가할 수 있다
- 스캐너 입력을 처리하는 방식:
  - 기존 레거시 PDA/POS 스캐너 로직이 있으면 그것을 우선 사용
  - 없으면 추상화 인터페이스 사용:
    - `BarcodeScannerGateway`
- 바코드로 상품 조회 성공 시:
  - 장바구니에 상품 추가
  - 이미 있으면 수량 증가
- 상품 미존재 시:
  - 사용자에게 토스트/스낵바/다이얼로그로 알림

### 7.7 상태 관리
- ViewModel에서 관리할 주요 상태:
  - `cartItems`
  - `selectedCategory`
  - `categoryProducts`
  - `totalAmount`
  - `discountAmount`
  - `receivedAmount`
  - `barcodeInputState`
- 수량 증가/감소/삭제/장바구니 할인/보류/전체취소는 모두 ViewModel 이벤트로 처리

### 7.8 Route / 용어
- Route:
  - `SUPER_HOME`
  - `PRODUCT_REGISTER`
- 여기서 `PRODUCT_REGISTER` 는 판매용 상품등록 화면을 의미한다

### 7.9 금지 사항
- ProductRegister 화면을 단순 폼형 등록 화면으로 구현하지 않는다
- ProductRegister 화면을 모바일 단일 컬럼으로 축소하지 않는다
- 우측 하단 시재/결제 액션 버튼 영역을 일반 리스트로 바꾸지 않는다
- 기존 Restaurant / Food 화면과 완전히 다른 스타일을 사용하지 않는다
